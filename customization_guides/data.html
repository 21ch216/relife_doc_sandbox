How to custom your databook

1. How to custom your databook#

The databook function instanciates a DataBook object. This functions take several arguments (see relife2.data) but it also allows extra key-words arguments like :

  • observed : Data instance (default to None)

  • left_censored : Data instance (default to None)

  • right_censored : Data instance (default to None)

  • interval_censored : IntervalData instance (default to None)

  • left_truncated : Data instance (default to None)

  • right_truncated : Data instance (default to None)

  • interval_truncated : IntervalData instance (default to None)

Thus, if one wants to use its own data format, it is possible by specifying one or all of the above arguments.

Data or IntervalData objects share a similar interface with two attributes

  • values (np.ndarray) : values of lifetimes, shape is always (n,) in Data and always (n,2) in IntervalData

  • index (np.ndarray) : index of lifetimes, shape is always (n,)

They are initialized thanks to a method called parse that must always return a tuple (index, values). Then, if a user wants to implement its own Data (or IntervalData object), he would write its own class like this :

from relife2.data.object import Data

class MyCustomData(Data)
    def __init__(self, *data):
        super().__init__(*data)

    def parse(self, *data):
        # a personal parsing process
        return index, values

After that, he could pass this new Data in the desired extra key-word argument of databook
